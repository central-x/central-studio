## 概述
魔方文档系统为了保证使用的简单性，没有提供过多的定制功能，需要文档编辑人员按照下文描述的逻辑，将文档、资源保存到合适的位置，才能正确显示文档。

魔方文档系统支持 2 种文档存放方式，别分别是 `本地文档库`、`Git 文档库`。

- 本地文档库: 文档系统保存在本地电脑的某个位置，然后在该位置下按本文描述的逻辑创建文档库。更新文档时，更新本地文档库里的文件即可。
- Git 文档库: 在 `本地文档库` 的基础上，将文档提交到 Git 源代码服务器中即可。更新时，可以在任意电脑上更新文件，更新完毕后提交变更即可。

## 创建文档库
文档编辑人员需要先在本地硬盘任意位置新建一个文件夹（文件夹名称需为英文且无特殊字符），并在该文件夹下创建以下目录结构。

```
document
 ├── 关于我们                          文档分类
 │   ├── layout.json                  分类目录结构描述文件
 │   ├── 测试文档1.md                  Markdown 文档
 │   ├── 测试文档2.md
 │   └── 二级目录                      文档目录
 │       ├── 测试文档3.html            HTML 文档
 │       └── 测试文档4.md
 ├── 产品说明                          文档分类
 │   ├── layout.json                  分类目录结构描述文件
 │   └── 协同办公.md
 ├── ${context-path}/doc/assets       文档资源存放目录
 ├── category.json                    文档分类说明
 └── document.json                    文档总体说明
```

### 文件和目录详解
- **${context-path}/doc/assets**

文档资源目录，主要用于存放文档的图片、压缩包等资源。该目录下只能支持再建两层目录，不能建多于两级的目录。**${context-path}** 需要替换成项目的上下文路径，默认为 `document`。

```
# 在Markdown中，可以这样访问资源，context-path替换成项目上下文路径
![](/${context-path}/doc/assets/logo.png)
```

- **document.json**

此文件用于描述文档的总体信息，主要需要提供 `版本`、`文档提供商名称`、`文档标题` 三个信息。

```json
{
   "version": "SNAPSHOT",
   "vendor": "广州明动软件股份有限公司",
   "title": "文档中心",
   "logo": "/${context-path}/doc/assets/logo.png"
}
```

> version 字段建议采用版本号 `v1.0.0` 或日期 `12/30` 这样的格式。
>
> logo 字段如果为空，则使用文档系统的默认图标，如果需要修改成指定的图标，可以将 logo 图片放到 `/${context-path}/doc/assets/` 目录下，然后在此文件指定该图片的路径即可。logo 图片的大小要求为 png 格式，30x30 大小。

- **category.json**

此文件用于描述文档的分类信息，每个分类信息都会显示到文档系统的导航栏中。

```json
// JSON 对象中，键为分类名称，值为分类英文名称
// 分类的英文名称推荐使用一个单词
{
    "关于我们": "about",
    "产品说明": "product"
}
```

- **layout.json**

此文件用于描述分类下面的文档布署结构。

```json
// JSON 对象中，键为文件名或文件夹名
// 如果键是文件夹，则值使用 JSON 对象表示
// 如果键是文件名，则值使用该文件名的英文翻译，推荐采用 word to word 规则，且值不能重复
// 整个 JSON 对象与目录结构一一对应
{
    "测试文档1.md": "test1",
    "测试文档2.md": "test2",
    "二级目录": {
        "测试文档3.html": "test3",
        "测试文档4.md": "test4"
    }
}
```
> 目前，文档系统暂时仅支持 .md 文件和 .html 文件。

## 创建版本化文档库
魔方文档中心支持版本化展示文档。在上面文档库的基础上，将产品说明文档进行版本化管理，只需要将原来的产品说明目录下的所有文件放入到对应的版本目录下即可。

```
document
 ├── 关于我们                          文档分类
 │   ├── layout.json                  分类目录结构描述文件
 │   ├── 测试文档1.md                  Markdown 文档
 │   ├── 测试文档2.md
 │   └── 二级目录                      文档目录
 │       ├── 测试文档3.html            HTML 文档
 │       └── 测试文档4.md
 ├── 产品说明                          文档分类
 │   ├── 1.0.x
 │   │   ├── layout.json                  分类目录结构描述文件
 │   │   └── 协同办公.md
 │   └── 1.1.x
 │       ├── layout.json                  分类目录结构描述文件
 │       └── 协同办公.md
 ├── ${context-path}/doc/assets       文档资源存放目录
 ├── category.json                    文档分类说明
 └── document.json                    文档总体说明
```

- **category.json**

修改该文件，添加产品的版本信息。

```json
// JSON 对象中，键为分类名称，值为分类英文名称
// 分类的英文名称推荐使用一个单词
{
    "关于我们": "about",
    "产品说明": {
        "code": "product",
        "versions": ["1.0.x", "1.1.x"]
    }
}
```

### 效果
完成以上步骤后，产品说明文档就可以通过版本化来展示了。
