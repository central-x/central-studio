## 1.0.x

## 概述
随着公司的业务发展以及政策的需要，客户对集约化的需求越来越多。这些需求对产品的性能、扩展性、可维护性等提出了很高的要求，因此对平台的要求也水涨船高。原有的基础资料平台[[链接](/public/doc#/basic/about)]无论是从性能、权限设计、数据存储模式、开发方式、灵活性等，都无法满业务部门快速开发的需求。同时，根据公司发展的需求，要求应用系统、平台软件需要采用 Spring Cloud 微服务设计架构，借此机会，平台部推出了魔方平台。

魔方平台是围绕着 Spring Cloud 微服务体系而设计、封装的一系列基础平台和方案，是明动软件所有应用的基础平台。魔方平台的目标是为公司所有产品提供一系列与业务无关的平台组件、提供应用的基础运行环境、提供灵活的技术解决方案，从而有效提升明动软件的技术竞争力。

开发者在接入和使用的过程中，如果遇到了问题、困难或修改建议，可以在搜索并加入公司钉钉群 30002275 沟通和交流。

## 组件简介
- [数据服务中心（mcube-provider）](/public/doc#/mcube/mcube-provider-about): 数据服务中心主要是为所有应用系统提供基础数据服务，包含租户类、组织架构类、权限类、系统数据，主要目的是提供通用的、业务无关的基础数据服务，解决『数据孤岛』问题，将各个应用系统有机地结合起来，形成整体技术解决方案。
- [用户认证中心（mcube-security）](/public/doc#/mcube/mcube-security-about): 用户认证中心主要是为所有应用系统提供基础用户认证功能。用户认证中心在使用的过程中，经常配合网关的用户认证插件一起使用，从而形成『准入网关』功能，保证内部应用系统的安全。通过用户认证中心，可以让用户登录一次即可使用所有业务系统的功能。
- [网关（mcube-gateway）](/public/doc#/mcube/mcube-gateway-about): 网关是微服务体系中『门户』，所有应用的请求，都需要经过网关。网关产品是基于 WebFlux 技术的自研产品，提供了数十种常用的过滤器插件、断言插件等，开发者可以通过可视化界面自由组合这些插件，为应用系统实现各类功能。
- [日志中心（mcube-logging）](/public/doc#/mcube/mcube-logging-about): 日志中心用于统一收集各个应用产生的各类日志，包括调试日志（deubg）、数据库日志(db)、网络日志（network）、访问日志（access）、执行日志（execution）、登录日志（login）、行为日志（action）等，可以极大的简化日志收集工作，为业务系统提供各类运营数据、为运维实施提供各类运行数据。
- [接口中心（mcube-gateway）](/public/doc#/mcube/mcube-api-about): 接口中心功能提供了简单易用的可视化界面，开发者可以通过这些可视化界面快速定义接口，供内部微服务系统或外部第三方公司使用。
- [存储中心（mcube-attachment）](/public/doc#/mcube/mcube-attachment-about): 存储中心主要是为所有应用系统提供了标准的文件存储的功能。本系统内部集成了插件系统，提供了简单易用的可视化界面。开发者通过这个插件系统，可以快速对接到各类第三方文件存储中心，如FastDFS、金山云存储等。存储中心内部封装了标准的上传、下载等接口，为各个应用系统提供集中式文件管理功能，可以有效提高项目的开发效率。
- [推送中心（mcube-push）](/public/doc#/mcube/mcube-push-about): 推送中心主要是为所有应用系统提供了标准的信息推送的功能。本系统内部集成了插件系统，提供了简单易用的可视化界面。开发者通过这个插件系统，可以快速对接到各类第三方推送服务，如站内信、邮件、短信、App 推送等。推送中心封装了标准模式的推送接口，可以忽略每种推送终端的特性，降低应用系统的接入难度；同时也提供了高级模式的推送接口，可以让应用系统精确控制每个终端的消息的展现形式。
- [门户（mcube-portal）](/public/doc#/mcube/mcube-portal-about): 门户系统主要是为用户提供了一个统一的应用入口，为不同的应用系统提供统一的支撑平台，实现业务展现、信息展示、应用整合、站内信等功能。
- [开放中心（mcube-open）](/public/doc#/mcube/mcube-open-about): 接口中心主要用于声明一些共公接口，供第三方业务系统接入。如第三方业务系统需要接入用户认证中心时，必须获取统一用户的相关信息，则第三方业务系统可以通过本系统获取相关组织架构数据。
- [文档中心（mcube-document）](/public/doc#/mcube/mcube-document-about):文档中心主要是提供了文档展示的功能。本系统不面向客户，主要是公司内部使用。本系统提供了简单易用的文档组织功能，通过 Git 保存文档，可以向公司的开发者提供在线阅读开发文档功能。
